# Presentation pedagogique et commerciale de l'application

Date : 2026-04-07

## 1. Objectif du document

Ce document sert de base pour construire une presentation professionnelle de l'application Mail Care.

Il est volontairement organise en deux parties :
- une presentation pedagogique et fonctionnelle, pour expliquer simplement comment l'application s'utilise ;
- une presentation commerciale, pour valoriser le produit, son utilite, son positionnement et son modele de service.

Le document s'adresse a plusieurs publics :
- personnes en situation de handicap ;
- personnes agees ;
- aidants familiaux ou professionnels ;
- associations ;
- entreprises ;
- collectivites ;
- partenaires techniques ou institutionnels.

## 3. Publics concernes

### 3.1 Personnes en situation de handicap

L'application est particulierement pensee pour les personnes qui rencontrent des difficultes a utiliser les outils mail classiques.

Exemples de situations concernees :
- mobilite reduite ;
- difficulte a utiliser le clavier ou la souris ;
- fatigue importante lors de la lecture ou de l'ecriture ;
- troubles visuels ;
- troubles cognitifs ou attentionnels ;
- besoin d'une assistance progressive et simplifiee ;
- besoin d'une interaction vocale.

Le but n'est pas de remplacer la personne, mais de lui redonner du controle et de l'autonomie.

### 3.2 Personnes agees

Le service s'adresse aussi aux personnes agees qui souhaitent continuer a utiliser leurs e-mails avec plus de confort.

L'application peut aider a :
- lire plus progressivement ;
- comprendre un message ;
- rediger une reponse ;
- eviter les erreurs de formulation ;
- utiliser la dictee ;
- reduire l'effort de manipulation.

### 3.3 Aidants, associations et professionnels

L'application peut etre utile dans un contexte d'accompagnement :
- aidants familiaux ;
- associations d'aide aux personnes handicapees ;
- services d'accompagnement ;
- structures medico-sociales ;
- collectivites ;
- entreprises engagees dans l'accessibilite numerique.

### 3.4 Entreprises

Pour les entreprises, l'application peut etre adaptee dans le cadre :
- de l'inclusion professionnelle ;
- du maintien dans l'emploi ;
- de l'amenagement de poste ;
- du traitement assiste de boites mail ;
- de contrats specifiques selon le nombre d'utilisateurs et les besoins.

## 4. Partie 1 - Presentation pedagogique fonctionnelle

### 4.1 A quoi sert l'application ?

Mail Care aide l'utilisateur a gerer ses e-mails avec moins d'effort.

L'application permet notamment de :
- connecter une boite mail ;
- lire ou analyser un message ;
- comprendre le contenu important ;
- classer ou prioriser les mails ;
- generer une reponse ;
- creer un nouveau mail ;
- modifier un texte ;
- dicter une phrase ou une correction ;
- utiliser une edition vocale assistee par IA ;
- ajuster l'affichage et l'accessibilite ;
- choisir le moteur IA utilise selon les besoins.

### 4.2 Principe general d'utilisation

Le parcours type est simple :

1. L'utilisateur se connecte a son compte.
2. Il ouvre la page mail.
3. Il connecte ou selectionne sa boite mail.
4. Il choisit le message ou le type d'action.
5. L'IA aide a comprendre, rediger ou corriger.
6. L'utilisateur garde le controle final avant envoi.

L'application ne vise pas a envoyer des messages sans controle humain.
Elle aide l'utilisateur a preparer, comprendre et valider.

### 4.3 Lire et comprendre un mail

Lorsqu'un mail est charge, l'application peut aider a :
- identifier le sujet principal ;
- extraire les informations importantes ;
- signaler ce qui manque ;
- aider a preparer une reponse ;
- prendre en compte des pieces jointes lorsque c'est possible ;
- adapter la reponse selon le contexte.

L'objectif est de reduire la charge de lecture et de comprehension.

### 4.4 Classer et prioriser les mails

Mail Care peut aussi aider a classer les mails pour reduire la charge de tri.

Le classement peut servir a :
- reperer les mails urgents ;
- distinguer information, demande d'action, reponse attendue ou message secondaire ;
- identifier les messages qui necessitent une validation humaine ;
- appliquer des regles de traitement ;
- prioriser les mails a lire en premier ;
- faciliter le suivi des messages deja traites.

Ce classement doit rester une aide a la decision.
L'utilisateur ou l'organisation garde la possibilite de verifier, corriger et adapter les categories.

### 4.5 Rediger une reponse

L'utilisateur peut demander une reponse assistee.

L'IA propose un brouillon.
L'utilisateur peut ensuite :
- le relire ;
- le corriger ;
- le reformuler ;
- le completer ;
- le valider ;
- le rejeter ;
- le supprimer.

Cette logique permet de conserver une validation humaine.

### 4.6 Creer un nouveau mail

L'application peut egalement aider a creer un message a partir d'une consigne.

Exemples :
- demander un rendez-vous ;
- envoyer une information ;
- repondre poliment a une demande ;
- rediger un mail administratif ;
- preparer un message professionnel.

L'utilisateur peut dicter ou saisir l'objectif du mail, puis l'IA propose un brouillon.

### 4.7 Dictee vocale

La dictee permet d'ecrire sans utiliser le clavier.

Elle peut servir a :
- inserer du texte au curseur ;
- remplacer une selection ;
- ajouter du texte a la fin ;
- reprendre apres un arret de session ;
- continuer une redaction en gardant ce qui a deja ete ecrit.

En cas d'indisponibilite de session de dictee, l'application peut revenir a un etat propre pour permettre de relancer.

### 4.8 Edition vocale assistee par IA

L'edition vocale assistee par IA est une fonction importante pour l'accessibilite.

Elle permet de donner des ordres vocaux du type :

```text
Dans Bonjour Madame, ajoute Mademoiselle virgule devant Madame.
```

ou :

```text
Positionne-toi en fin de page et ajoute le mot cordialement.
```

L'objectif est d'eviter la combinaison difficile :
- positionnement manuel ;
- selection ;
- puis dictee.

Avec l'edition vocale, l'utilisateur peut exprimer directement l'action a realiser.

Cette fonction est particulierement utile pour les personnes qui peuvent parler, mais qui ne peuvent pas utiliser facilement leurs mains, un clavier, une souris ou un systeme de suivi oculaire.

### 4.9 Utilisation sans parole

Mail Care ne doit pas etre presente comme une application uniquement vocale.

Certaines personnes handicapees peuvent etre privees de la parole, avoir une parole difficilement reconnaissable, ou ne pas pouvoir utiliser la voix de maniere fiable.

Dans ces situations, l'application doit rester utile grace a d'autres modes d'interaction :
- saisie clavier classique ou adaptee ;
- clavier virtuel ;
- contacteur ou dispositif d'assistance compatible avec la navigation web ;
- selection progressive ;
- lecture progressive ;
- boutons agrandis ;
- confirmations renforcees ;
- choix dans des listes ou suggestions ;
- prompts courts prepares par l'utilisateur ou l'aidant ;
- copier-coller depuis un outil de communication alternative ;
- utilisation avec un systeme externe d'aide a la saisie.

Le principe important :

```text
La voix est un accelerateur d'accessibilite, mais elle ne doit pas etre le seul mode d'acces.
```

Pour les personnes privees de la parole, l'objectif est de proposer une interaction plus lente mais fiable, avec moins de gestes complexes et plus de guidage.

### 4.10 Parametres d'accessibilite

L'application prevoit des reglages pour adapter l'experience :
- taille de texte ;
- police ;
- espacement ;
- largeur de lecture ;
- lecture progressive ;
- lecture audio ;
- boutons plus grands ;
- confirmations renforcees ;
- choix de la dictee ;
- choix du fournisseur IA.

L'objectif est de permettre une utilisation confortable et personnalisable.

### 4.11 Confidentialite et donnees

Le principe retenu est que l'application n'a pas vocation a conserver les mails complets.

Les mails restent geres dans chaque boite mail individuelle.
L'application conserve uniquement les informations necessaires au fonctionnement :
- compte ;
- preferences ;
- connexions ;
- droits ;
- traces d'action utiles ;
- configuration technique.

Une page RGPD permet de presenter les principes de conservation et d'exporter les donnees du compte connecte.

### 4.12 Role de l'utilisateur

L'utilisateur reste au centre du processus.

Il peut :
- choisir le mail a traiter ;
- lancer une generation ;
- modifier le brouillon ;
- valider ;
- rejeter ;
- supprimer ;
- relancer une dictee ;
- changer les parametres.

L'application assiste, mais ne doit pas deposseder l'utilisateur de sa decision.

Pourtant, elle reste difficile d'acces pour de nombreuses personnes :
- interfaces denses ;
- lecture longue ;
- nombreuses manipulations ;
- difficultes de saisie ;
- fatigue cognitive ;
- besoin de precision ;
- stress lie a l'ecriture ;
- risque d'erreur avant envoi.

Pour les personnes en situation de handicap ou les personnes agees, ces obstacles peuvent reduire l'autonomie numerique.

Mail Care apporte une reponse concrète : rendre l'e-mail plus simple, plus vocal, plus progressif et plus assiste.

### 5.2 Proposition de valeur

Mail Care combine :
- intelligence artificielle ;
- dictee vocale ;
- edition vocale ;
- classement assiste des mails ;
- interface accessible ;
- controle humain ;
- personnalisation des reglages.

Le produit aide l'utilisateur a :
- gagner en autonomie ;
- reduire l'effort de lecture et d'ecriture ;
- produire des reponses plus claires ;
- utiliser son mail avec moins de manipulations ;
- garder la main sur les decisions importantes.

Le service est gratuit pour :

Pour les autres utilisateurs individuels :
